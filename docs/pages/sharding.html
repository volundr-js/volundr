Sharding

Scale your bot across multiple WebSocket connections.

Auto sharding

Völundr uses worker thread sharding - more memory-efficient than process-based sharding.

const client = new Client({
    token: "...",
    intents: ...,
    shardCount: "auto",  // Discord tells us how many shards
});

Manual shard count

const client = new Client({
    token: "...",
    intents: ...,
    shardCount: 4,
});

Thread pool

For fine-grained control over worker threads:

import { ThreadPool } from "@volundr/client";

const pool = new ThreadPool({
    workerPath: "./worker.js",
    size: 4,
});
When to shard: Discord requires sharding at 2,500 guilds. Below that, a single connection is fine.
