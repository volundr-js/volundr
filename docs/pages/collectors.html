Collectors

Collect messages, button clicks, and reactions over time.

Message collector

const collector = channel.createMessageCollector({
    filter: (msg) => msg.author.id === userId,
    time: 30_000,
    max: 5,
});

collector.on("collect", (msg) => {
    console.log(msg.content);
});

collector.on("end", (collected) => {
    console.log(`Got ${collected.size} messages`);
});

Component collector

const collector = message.createComponentCollector({
    filter: (i) => i.user.id === userId,
    time: 60_000,
});

collector.on("collect", (interaction) => {
    if (interaction.customId === "next") {
        interaction.update({ content: "Page 2" });
    }
});

Await shorthand

// Wait for one button click (30s timeout)
const clicks = await message.awaitComponents({
    time: 30_000,
    max: 1,
});

Reaction collector

const collector = message.createReactionCollector({
    filter: (reaction, user) => reaction.emoji.name === "👍",
    time: 60_000,
});

collector.on("collect", (reaction, user) => {
    console.log(`${user.username} reacted!`);
});

Automatic cleanup with using

Collectors implement Symbol.dispose, so you can use the using keyword for automatic cleanup:

// Collector is automatically stopped when it goes out of scope
{
    using collector = channel.createMessageCollector({
        filter: (msg) => msg.author.id === userId,
        time: 30_000,
    });

    collector.on("collect", (msg) => {
        console.log(msg.content);
    });
}
// collector.stop("dispose") is called automatically here
Note: The using keyword requires TypeScript 5.2+ with "lib": ["esnext"]. It guarantees the collector is stopped even if an exception is thrown.
