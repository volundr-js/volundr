Why it's fast

Event pipeline

WebSocket → JSON.parse → _patch() → cache → emit

Zero intermediate copies. Entities are patched in-place - MESSAGE_UPDATE only touches changed fields.

Zero-allocation design

  • Entities reuse objects. _patch() mutates in place instead of creating new instances.
  • Lazy initialization - Date objects and Set collections are only created when first accessed.
  • 2.4× less GC garbage than Node.js EventEmitter under sustained load.

Memory efficiency

  • 259 bytes per cached member (vs 400+ in typical implementations).
  • Configurable sweepers prevent unbounded cache growth.
  • Worker thread sharding shares the V8 heap more efficiently than child processes.
