Völundr

A Discord SDK for TypeScript. Fast, predictable, zero magic.

Node.js 18+  •  Discord API v10  •  TypeScript 5+

Why Völundr?

Predictable API. .get() reads from cache. .fetch() calls the API. .send() accepts the same input everywhere. You can guess the method before opening docs.

No abstraction hell. Three layers: gateway, REST, entities. No 20 intermediate managers. You always know what's happening.

Fast where it matters. 500k+ events/sec. Sub-microsecond latency. 2.4× less GC garbage than Node.js EventEmitter. But you'll never need to think about this - it just doesn't lag.

Scales. Cache sweepers, configurable limits, worker thread sharding, global rate limit coordination. Built for bots that run for months on hundreds of shards.

Mental Model

Discord API
     │
  Gateway ──→ events (WebSocket, shards, compression)
  REST    ──→ requests (rate limits, retries, buckets)
     │
  Client  ──→ entities + cache + builders
     │
  Your bot

Three layers. Gateway handles WebSocket events. REST handles HTTP requests. Client ties them together with rich entities, cache, and builder DSL.

What makes it different

Not a discord.js clone. Different design decisions from the ground up.

  • Declarative routing. client.command("ping", handler) and client.button(/^ticket-\d+$/, handler) instead of manual if/switch on interaction names and custom IDs.
  • Modern resource management. await using client = new Client(...) - automatic cleanup via Symbol.asyncDispose. Collectors support Symbol.dispose.
  • Event streaming. for await (const msg of client.stream("MESSAGE_CREATE")) - consume events as async iterables with AbortSignal cancellation.
  • Tagged template mentions. fmt`Hello ${user}!` auto-resolves entities to Discord mention syntax.
  • Integrated builders. Functional helpers (embed(), row(), button()) and callback DSL (msg => { msg.embed(...) }) ship with the client - no separate package needed.
  • Built-in thread pool. ThreadPool for multi-core shard processing using worker threads, no external sharder process.
  • TypeScript-first. Type predicates (interaction.isChatInputCommand()), typed events, generics throughout. Not types bolted onto JavaScript.
  • Single import. @volundr/client includes everything - entities, builders, cache, collectors, permissions, formatters. Most bots need one dependency.
