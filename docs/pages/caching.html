Caching

How Völundr manages data in memory.

How cache works

Völundr caches entities automatically as events arrive from the gateway. The cache is the single source of truth for your bot.

The two rules

  • .get(id) - reads from cache. Instant, never hits the API.
  • .fetch(id) - calls the Discord API. Updates cache with the response.
// From cache (instant, may be null)
const guild = client.guilds.get("123456");

// From API (always fresh, updates cache)
const guild = await client.fetchGuild("123456");

Collection

All caches use Collection, an extended Map with 25+ utility methods:

// Filter, map, find
client.guilds.filter(g => g.memberCount > 100);
client.users.find(u => u.username === "Alice");
guild.members.map(m => m.user.username);

// first, last, random
guild.roles.first();
guild.members.random();

// sort, partition, reduce
guild.roles.sort((a, b) => b.position - a.position);

LimitedCollection

For memory-sensitive caches, use LimitedCollection with a max size and eviction:

import { LimitedCollection } from "@volundr/client";

const cache = new LimitedCollection({ maxSize: 1000 });
